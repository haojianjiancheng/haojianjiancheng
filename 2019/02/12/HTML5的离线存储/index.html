HTML5的离线存储

HTML5的离线存储

二月 12, 2019

起源：

在HTML5之前的网页，必须有联网才能访问，这也是WEB的一个特色。这对于PC的时代影响不大，但到了移动移动互联网时代，设备不固定，而且依赖无线信号。网络的可靠性降低，很多因素会造成断网，这对于WEB的伤害很大。因此，HTML5离线存储（Cache Manifest）诞生。

什么是Cache Manifest：

  • manifest是一个后缀名为minifest的文件。文件中定义需要离线存储的文件。支持manifest的浏览器，会按照manifest文件的规则，将文件保存在本地，从而在没有网络链接的情况下，也能访问。
  • 当我们第一次配置manifest的文件后，再次访问网页，浏览器就会检测manifest文件是否有变动。如果变动就会把相应的更新下载，同时改变浏览器里的manifest文件。如果没有变动，就会把manifest文件返回。

Manifest的特点：

  • 离线浏览：用户可以在离线状态下浏览网站内容
  • 更快的速度：因为数据被存储在本地，所以速度更快。
  • 减轻服务器的负载：浏览器只会下载服务器上发生改变的资源。

Manifest的缺点：

  • 更新完版本后， 必须刷新一次才会启动新版本。
  • 进入离线缓存页面，如果不更新版本，会将其当成静态页面不请求。
  • 无法进行灰度发布等策略。
  • 无法增量更新。

与传统浏览器缓存的区别：

  • Manifest针对整个应用，浏览器缓存是单个文件
  • Manifest断网依然可以打开页面，浏览器缓存不行。
  • Manifest可以主动通知浏览器更新资源。

浏览器支持：

IE10+、Firefox3+、Safari4+、Opera10.6、Chrome、iOS3.2+、Android Webkit

后面以#开头的是注释，一般会在第二行写个版本号，用来在缓存的文件更新时，更改manifest的作用，可以是版本号，时间戳或者md5码等等。
manifest文件，基本规格分为三段：

  • CACHE(必须)——离线资源
    标出哪些文件需要缓存，可以是相对的路径，也可以是绝对路径。
  • NETWORK(可选)——联网资源
    不缓存的文件，离线不可用。可以用*来只是除了CACHE的文件，其他都需要联网。
    这里要注意的是，如果这里没有用*，而是用了具体的地址，那么只要不在CACHE和NETWORK的文件都报加载失败。即使联网了也一样。
  • FALLBACK（可选）——替代资源
    指定后备页面，当资源无法访问时，浏览器会使用该页面。

第一个url表示资源，第二个表示后备页面。这两个url必须都是相对路径，并且与清单文件同源，可以使用通配符。

用404.html代替所有html文件。

如何更新缓存：

  • 更新manifest文件
    给manifest文件添加或删除文件，都会更新缓存
    通过更改版本号，可以很好的更新manifest文件。

  • 通过JS操作

  • 清除浏览器缓存
    如果用户清除了浏览器缓存（手动或者自动）都会从新下载文件。

相关JS API：

核心是applicationCache对象，有个status属性：

  • 0 （UNCACHED）：无缓存，即没有与页面相关的应用缓存。
  • 1（IDLE）：闲置，即应用缓存未得到更新。
  • 2（CHECKING）：检查中，即应用缓存正在下载描述文件中指定的资源。
  • 3（DOWNLOADING）：下载中，即应用缓存正在下载描述中指定的资源。
  • 4（UPDATEREADY）：更新完成，所有资源都已经下载完毕。
  • 5（IDLE）：废弃，即应用缓存的描述文件已经不存在了，因此页面无法访问应用缓存。

相关事件：

  • checking：在浏览器应用缓存查找更新时触发。
  • error：在检查更新或下载资源期间发送错误时触发。
  • noupdate：在检查描述文件发现文件无变化时触发。
  • downloading：在开始下载应用缓存资源时触发。
  • progress：在文件下载应用缓存的过程中持续不断的下载地触发
  • updateready：在页面更新的应用缓存下载完毕触发。
  • cached：在应用缓存完整可用时触发。

注意事项：

  • 浏览器对缓存数据的容量限制可能不一样（一些浏览器限制每个站点5MB）
  • 如果manifest文件，或者内部列举的某个文件不能正常下载，则整个更新过程失败。浏览器继续全部使用旧的缓存。
  • 引用manifest的html必须与manifest文件同源，在同一个域下。
  • FALLBACK中的资源必须和manifest文件同源。
  • 当一个资源被缓存后，该浏览器直接请求这个绝对路径也会访问缓存中的资源。
  • 站点中的其他页面即使没有设置manifest属性，请求的资源如果在缓存中，也从缓存中访问。
  • 当manifest文件发生改变时，资源请求本身也会触发更新。
  • manifest文件需要配置正确的MIME-type，即 “text/cache-manifest”，必须在web服务器上进行配置。
  • 更新完版本后，必须刷新一遍页面才能启动新版本。
